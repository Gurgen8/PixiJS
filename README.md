# Space Shooter

## Описание проекта

Проект представляет собой браузерную игру «Space Shooter», разрабатываемую с использованием библиотеки **Pixi.js (v8)** и языка **TypeScript**. На данный момент заложен надежный архитектурный фундамент игры, который готов к дальнейшему расширению и добавлению игровой логики. 

### Что уже реализовано:

1. **Ядро игры (Engine & App)**
   - Настроена инициализация приложения Pixi.js с использованием актуального API v8.
   - Реализован паттерн Singleton для ключевых классов (`Game`, `App`), что обеспечивает единый доступ к ресурсам игры из любой части кода.
   - Настроен Game Loop (игровой цикл) с привязкой к `Ticker` для обновления состояния игры.
   - Внедрена система **Responsive Canvas** (адаптивного холста) — игра автоматически масштабируется под размер окна браузера с сохранением оригинальных пропорций (метод letterboxing/pillarboxing).

2. **Менеджеры (Managers)**
   - **SceneManager**: Система управления сценами. Позволяет легко переключаться между состояниями игры (например, меню, сам геймплей, экран проигрыша) и передавать им события обновления (`update`) и изменения размера (`resize`).
   - **AssetManager**: Подготовлена структура для загрузки и кэширования игровых ресурсов (текстуры, звуки).
   - **InputManager**: Архитектура для обработки пользовательского ввода (клавиатура/мышь).

3. **Система событий (EventBus)**
   - Реализована глобальная шина событий (`EventBus`), позволяющая различным компонентам игры общаться между собой без жесткой связности (паттерн Publish-Subscribe).
   - Описаны базовые константы событий в `GameEvents` (добавление очков, урон игроку, конец игры).

4. **Игровые сцены (Scenes)**
   - Создан базовый класс сцены `BaseScene`.
   - Реализована стартовая сцена `MainMenuScene` с пульсирующим текстом заголовка («SPACE SHOOTER»), демонстрирующая работу игрового цикла и отрисовку текста средствами Pixi.js.

Таким образом, проект имеет полностью рабочий каркас: настроены типы, сборка (Vite), адаптивный рендеринг и система управления игровыми состояниями. Можно приступать к добавлению конкретных игровых сущностей (корабль игрока, враги, снаряды) и механик.
