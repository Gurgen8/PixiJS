# Pixi.js Game Engine Project

## Обзор архитектуры
Этот проект представляет собой модульный игровой движок на базе **Pixi.js (v8)** и **TypeScript**, который объединяет несколько мини-игр в одном приложении. 

### Основные компоненты (Core & Managers):
1. **Core**: `Game` / `App` - инициализация Pixi.js, настройка адаптивного холста (Responsive Canvas) и игрового цикла (Game Loop).
2. **Менеджеры (Managers)**:
   - **SceneManager**: Управляет жизненным циклом сцен (переключение, `update`, `resize`).
   - **AssetManager**: Загрузка и кэширование ресурсов (спрайты, звуки).
   - **InputManager**: Глобальная обработка ввода (клавиатура, мышь).
   - **AudioManager**: Управление звуковыми эффектами и фоновой музыкой.
3. **Система событий**: Глобальная шина событий (`EventBus`) для связи между компонентами без жесткой связности.
4. **Сцены (Scenes)**: Базовый класс `BaseScene` для всех сцен, а также `MainMenuScene` с анимированным главным меню и списком игр.

### Структура игр (Games)
Каждая игра располагается в отдельной директории внутри `src/games/` и следует собственной архитектуре, основанной на принципах компонентного подхода и разделения ответственности:
- **`GameManager.ts`**: Центральный контроллер игровой логики (очки, жизни, уровни/волны, состояние игры).
- **`entities/`**: Игровые сущности и объекты (Player, Enemy, Bomb и т.д.).
- **`systems/`**: Логические системы (например, EnemySpawner, ItemSpawner).
- **`scenes/`**: Специфичные для игры сцены (например, GameScene).
- **`ui/`**: Локальный интерфейс (HUD).

В проекте реализованы 3 игры:
- [Bomberman](./src/games/bomberman/README.md)
- [Fruit Collector](./src/games/fruitcollector/README.md)
- [Space Shooter](./src/games/spaceshooter/README.md)

## Как запустить

1. Установите зависимости:
```bash
yarn install
```

2. Запустите сервер разработки:
```bash
yarn dev
```
